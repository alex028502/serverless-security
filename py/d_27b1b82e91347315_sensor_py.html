43# I got a lot of false alarms from a single PIR sensor, so I put two PIR 

44# sensors instead. They both have to be on to trigger the alarm. This program 

45# aggregates the results, and sends it to another GPIO output so that that can 

46# be used to activate the main program (by 'pushing a button') that takes a 

47# picture and sends the encrypted email. I used GPIO so that I could manually 

48# debug the connection between the two subsystems, because there is no 

49# automated test doing that. 

50 

51# I think I have this problem 

52# https://forums.raspberrypi.com/viewtopic.php?t=254103 except that one is 

53# happening every 16 minutes and mine is happening every 5 minutes. 

54# Once I put two of them, it almost never happens (but still happens sometimes). 

55# I guess the odds of both PIR sensors finding themselves in the path of the 

56# thing that happens every 5 minutes and sets them off is quite slim. If I 

57# logged the sensor input, I could catch it doing it with one of the PIR 

58# sensors and not the other, but I haven't done that. 

59 

60# Initially when I decided I needed two PIR sensors, I connected them to two 

61# relay modules, and then put the output of the relay modules in series. That 

62# worked great, until it stopped working all of a sudden. Then I realised that 

63# these were five volt relay modules. I ordered some 3V relay modules... that 

64# almost don't exist. I replaced my relays with the 3V ones, and that didn't 

65# work either. Then I read somewhere that one of the kinds of 3V relay modules 

66# I got was flawed, and that it still requires 5V because of an optocoupler. So 

67# instead of using relays in series, I thought it would be quite easy to do with 

68# an arduino... except then I would have to figure 

69# out how to test the arduino program, or it would just seem like a way to 

70# avoid testing, and it would be twice as hard to deploy. Then I found out what 

71# I always assumed was not the case, that two different programs on the same 

72# raspberry pi can use different GPIO pins. So I replaced the two relays with 

73# their outputs in series, and their inputs connected to the PIR sensors with 

74# a second rpi program (this one), but still connected it to the input of the 

75# existing program using GPIO as though it is an external device, and I did 

76# that with a 3V relay so that I didn't risk connecting an output pin to an 

77# input pin incorrectly. I used the kind of 3V relay that didn't have the 

78# supposed 5V optocoupler flaw in it, and that worked great! ... until it 

79# stoppped working, and the LEDS on the relay module went on but the circuit 

80# didn't close. So then I asked a question on stack exchange and got over 

81# my fear of connecting two GPIO pins together, and removed the relay from the 

82# equaion, having learned that relay modules have their own nuances. 

83
